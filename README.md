# AI-Powered Customer Inquiry Assistant for a Textile Company

## Project Overview

This project is a prototype of an AI-powered customer inquiry assistant for a textile company. The goal is to help potential international customers ask questions about the company’s products, services, sustainability focus, production support, pricing discussion, shipping discussion, and inquiry process.

The assistant uses a simple Retrieval-Augmented Generation (RAG) workflow. Instead of answering only from general AI knowledge, the system first retrieves relevant company information from a structured knowledge base and then generates a grounded answer based on that retrieved context.

This project was created for a Generative AI for Business Analytics course. It demonstrates how company knowledge, chunking, retrieval, and grounded answer generation can be used in a practical business use case.

---

## Business Problem

International customers may have questions about textile products, denim fabric, customization, quality control, sustainability, pricing, shipping, and production support. However, customers may not always know where to find this information on a company website or how to ask the right questions.

This project explores whether a RAG-based assistant can help answer customer inquiries clearly and consistently using company knowledge content.

## Technical Design

The project follows a simple RAG-style workflow:

1. Load the company knowledge base from company_info_sample.txt.
2. Split the knowledge base into topic-based sections.
3. Store the sections in a pandas DataFrame.
4. Convert the section text into TF-IDF vectors.
5. Convert the user question into a TF-IDF vector.
6. Use cosine similarity to retrieve the most relevant section.
7. Generate a grounded answer using the retrieved context.
8. If the similarity score is too low, return an out-of-scope response.

## How to Run the Interactive Demo

In the terminal, run:

bash python3 rag_final.py 

The program will ask for a customer question.

Example question:

text Do you support recycled denim fabric? 

The system will return:

- Best matched section
- Similarity score
- Retrieved context
- Final grounded answer

To stop the program, type:

text exit 

---

## How to Run Batch Testing

In the terminal, run:

bash python3 batch_test.py 

The script will test all 100 questions from sample_questions.csv and generate:

text batch_test_results.csv 

The batch test reports:

- Number of knowledge sections loaded
- Number of questions tested
- Exact section match accuracy
- Scope detection accuracy

Current test result:

text Knowledge sections loaded: 45 Questions tested: 100 Exact section match accuracy: 57.00% Scope detection accuracy: 99.00% 

---

## Evaluation Notes

The exact section match accuracy is strict because it only counts a result as correct when the predicted section exactly matches the expected section.

Some questions may have more than one reasonable matching section. For example, a question about custom fabric development may reasonably match Customization Support, Fabric Development, or FAQ - Fabric Development Questions.

Because of this, the exact section match score should be interpreted as a strict retrieval benchmark rather than the full quality of the assistant.

The scope detection result is strong because the assistant correctly identifies most in-scope and out-of-scope questions.

## Limitations

This project is still a prototype.

Current limitations include:

- The company data is sample data rather than a full real company database.
- TF-IDF is useful and explainable, but it may not capture deeper semantic meaning as well as modern embedding models.
- The answer generation is simple and grounded in retrieved text, but it is not yet connected to a production-level LLM.
- The current interface runs in the terminal instead of a customer-facing website.
- The evaluation uses exact section matching, which may be too strict when multiple sections are semantically reasonable.

---

## Future Improvements

Future improvements could include:

- Adding more real company documents and website content.
- Using stronger embedding models.
- Using a vector database such as FAISS, Chroma, or Pinecone.
- Connecting the retrieved context to a large language model for more natural responses.
- Building a simple web interface for customers.
- Improving evaluation with more flexible semantic matching instead of exact section matching only.

## Project Summary

This project demonstrates a practical RAG-based customer inquiry assistant for a textile company. It shows how structured company knowledge can be chunked, retrieved, and used to generate grounded responses for customer questions.

The current prototype includes a structured knowledge base, 100 sample questions, an interactive terminal demo, and a batch testing evaluation workflow.
